# Task 2: 页面结构重构 + 阅读模式 + 后台感清除

## 目标
文章页改为 Reader Mode 结构，从"后台管理器"升级为"沉浸式阅读器"。

## 项目路径
`/home/flypigs/typecho-manager-compose`

## 1. 顶部 Header 极简化
**现状问题**：信息太多，太像后台
**改为**：
- 左：返回按钮（IconButton, arrow_back）
- 中：标题（1行截断，ellipsis, maxLines: 1）
- 右：更多菜单（IconButton, more_vert）

**禁止出现**：
- ❌ 作者信息
- ❌ 标签堆叠
- ❌ 时间文字
- ❌ 任何 metadata 信息

## 2. Cover 封面区（独立区域）
- 如果有封面图：全宽显示
- 宽高比：16:9 或 4:3
- 圆角：12~16
- 底部渐变遮罩（40%高度，从透明到黑色半透明）
- **无封面则完全隐藏**（不要留空白占位）

## 3. Meta 信息区（统一单行）
替换现在混乱的多行信息，改为**一行**：
```
05-30 · 8 min read · #生活杂谈
```
- 日期（MM-dd 格式）
- 阅读时间（自动估算：总字数 / 300 = 分钟数）
- 分类（带 # 前缀）
- 用 " · " (middle dot) 分隔
- **不要卡片化**，纯文本即可
- 字体：bodySmall 或 12~13sp，颜色：灰色

## 4. 阅读模式开关（Reader Mode Toggle）
- 在 Header 的更多菜单（…）中添加"阅读模式"选项
- 开启后：
  - 隐藏所有非正文 UI（Header 简化为只有返回按钮和标题）
  - 隐藏 Meta 信息区
  - 隐藏评论区、统计信息
  - 只保留正文内容
  - 背景可以略微调暗或调暖（护眼）
- 类似 Safari Reader Mode
- 再次点击可退出阅读模式

## 5. 清除后台感 UI（重要！）
**必须删除/隐藏的元素**：
- ❌ 卡片式 metadata 展示（作者头像+名字+时间的卡片）
- ❌ 密集统计信息（阅读数、评论数等堆叠）
- ❌ debug 风格排版（等宽字体显示普通信息、代码风格的布局）
- ❌ 后台管理器风格的工具栏

**替换为**：
- ✅ 纯阅读优先的布局
- ✅ 信息弱化（meta 用小字灰色，不抢视觉焦点）
- ✅ 控件弱化（操作按钮用 IconButton，不用 FAB 或大按钮）

## 6. 正文区域
- 纯阅读模式
- 不要卡片包裹正文
- 背景色跟随页面背景（不要单独的白色卡片）

## ⚠️ 注意事项
- 不要删掉文章加载功能
- 不要删掉图片显示功能
- 封面图和 meta 信息只是换了布局，不是删了
- 改完必须编译验证通过
- 不要 git push
